Documentação da 1ª Sprint – Projeto MaternarFoco: Saúde Preventiva da Gestante .(Prevenção da Prematuridade)Data: 27/03/20261. Definição do Escopo e RequisitosEscopo do Projeto:Desenvolvimento de uma aplicação móvel focada na saúde preventiva de gestantes. O sistema utilizará análise de dados históricos do DATASUS para agrupar perfis de risco gestacional. Através de um questionário interativo no aplicativo, a plataforma classificará a usuária em um desses perfis, fornecendo orientações de saúde personalizadas, alertas nutricionais e guias de acompanhamento pré-natal.Requisitos Funcionais (RF):RF01: O sistema deve permitir o cadastro e a autenticação segura de usuárias (gestantes).RF02: O aplicativo deve apresentar um questionário dinâmico coletando dados sociodemográficos e clínicos (idade, escolaridade, peso, altura, histórico de internações e quantidade de consultas pré-natal).RF03: O sistema deve calcular o perfil de risco da gestante utilizando a distância matemática em relação aos clusters de referência.RF04: O aplicativo deve exibir uma tela "Seu Perfil", apresentando dicas preventivas e alertas baseados no grupo (cluster) da usuária.Requisitos Não Funcionais (RNF):RNF01: A API backend deve responder ao cálculo de classificação de perfil em menos de 500ms.RNF02: O aplicativo móvel deve operar em ambiente multiplataforma (Android e iOS).RNF03: O tráfego de dados sensíveis de saúde deve ser criptografado ponta a ponta (HTTPS).2. Modelagem InicialArquitetura do Sistema: Padrão Cliente-Servidor dividido em três frentes:Front-end: Interface móvel interativa.Back-end (API Restful): Lógica de negócios, cálculo de classificação e gerenciamento de usuários.Pipeline de Dados: Ambiente analítico para processamento do DATASUS e definição dos parâmetros do modelo de inteligência.Caso de Uso Central: Gestante realiza login $\rightarrow$ Preenche dados de saúde atuais $\rightarrow$ API cruza informações com os centróides pré-calculados $\rightarrow$ App renderiza jornada de cuidados específica.3. Criação do Repositório (GitHub)Estrutura: Organização criada no GitHub contendo repositórios isolados para mobile-app, api-core e data-pipeline.Versionamento: Adoção do fluxo Git Flow, mantendo a ramificação main para código de produção e develop para integração contínua das sprints.4. Estrutura Inicial do Back-endFramework: NestJS (TypeScript).Configuração da Sprint: Projeto inicializado. Módulos, controladores e serviços estruturados para os domínios Users, Auth e RiskClassification.Integração: ORM configurado e preparado para a persistência de dados no banco relacional.5. Protótipo Inicial do Front-endFramework: Flutter (Dart).Entregáveis (Telas Estáticas):Onboarding e Autenticação.Home (Visão geral da gestação).Assessment (Questionário de fatores de risco e histórico clínico).Dashboard de Saúde (Resultados do perfil e trilha de cuidados).6. Banco de Dados ModeladoTecnologia: PostgreSQL.Modelagem Lógica Inicial:usuarios: Dados de credenciais e identificação.dados_gestacionais: Histórico de inputs do aplicativo (idade, IMC, consultas realizadas, comorbidades relatadas).clusters_referencia: Armazenamento das coordenadas matemáticas (centróides) e metadados de cada perfil de risco mapeado.7. Computação em Nuvem IIInfraestrutura: Servidor Virtual Privado (VPS) gerenciado via EasyPanel.Justificativa: A orquestração de containers Docker pelo EasyPanel automatiza o deploy da API em NestJS e do banco PostgreSQL. Isso garante provisionamento rápido, facilidade na gestão de certificados SSL e escalabilidade do ambiente, permitindo que a equipe foque na regra de negócio e não na administração bruta da infraestrutura.8. Mineração de Dados (Data Science)Ambiente de Desenvolvimento: Google Colab (Python).Bases de Dados (Ecossistema DATASUS): A mineração adotará uma abordagem multicêntrica para garantir a precisão do algoritmo de Clustering (K-Means ou Hierárquico via scikit-learn), gerando de 4 a 6 grupos de risco:SINASC (Nascidos Vivos): Base primária para extração de variáveis sociodemográficas (idade materna, escolaridade, histórico de gestações).SISVAN (Nutricional): Integração de dados de IMC e curvas de ganho de peso, fundamentais para prever riscos de prematuridade e pré-eclâmpsia.SIH/SUS (Hospitalar): Mapeamento do impacto de internações prévias por comorbidades gestacionais (ex: infecções, picos hipertensivos mapeados pelo CID-10).SIA/SUS (Ambulatorial): Análise do volume e complexidade das consultas de pré-natal, identificando perfis de desassistência.SIM (Mortalidade): Cruzamento histórico para identificar os padrões demográficos e clínicos associados aos desfechos mais graves, formando os parâmetros do "Cluster Crítico".
